  What was done

  Custom art as defaults (BootScene fix)

  Previously, every boot would re-process bundled images and store them in localStorage, which could hit Android
  WebView's ~5MB storage quota causing images to silently fail and fall back to procedural graphics. Also, any
  images a player uploaded via the Avatars/Backgrounds menu were overwritten on the next restart by the bundled art.

  Now:
  - preload() checks localStorage first — if a player has uploaded a custom image for a slot, it loads that;
  otherwise loads the bundled custom-art/*.png
  - _processBundledTextures() resizes/flips the bundled art in memory but never writes to localStorage — the
  _bundled set serves as the flag for hasCustom(), zero quota used
